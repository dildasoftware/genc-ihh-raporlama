 0. Yaklaşım — Önce Doğru Çalışsın
Ana Prensip
Canlıya alma zorunluluğu yok. Önce her şey lokal ortamda doğru mantıkla çalışsın.
Jüriye lokalden de demo verebiliriz. Bu bize daha fazla özellik geliştirme alanı sağlar.
Kesinti kuralı: hiçbir veri, rapor, karne veya AI çıktısı otomatik silinmez. Yalnız kullanıcı elle silerse silinir.
0.1 Bu Belgenin Kapsamı
Bu belge, önceki iki plandaki eksikleri gidererek nihai halini alır. Şu güncellemeleri içerir:
•	Cinsiyet mantığı düzeltmesi: varsayılan görünüm birleşik (K+E toplamı), filtreyle ayrıştırılır.
•	Genel Merkez birim başkanı rolü eklendi (ör. Üniversite Başkanı → tüm illerin üniversite verisi).
•	Bölge koordinatörü cinsiyet ayrımı yapmaz, her iki kolu da görür ve karşılaştırır.
•	PDF arşivleme sistemi: haftalık raporlar ve karneler PDF olarak üretilir, tarihli olarak saklanır.
•	AI, haftalık il raporlarını da üretebilir (opsiyonel).
•	İl koordinatörü kendi birimleri arasında (üniversite/lise/ortaokul/çocuk) karşılaştırma yapabilir.
•	Kalıcılık ve arşiv katmanı sistemin temel bir parçası, sonradan eklenen özellik değil.
 1. Rol ve Yetki Matrisi (Kesin)
1.1 Rol Listesi
Rol	Cinsiyet Bağlantısı	Görebildiği Veri	Yapabildiği
İl Koordinatörü (K)	Sadece kadın kolu	Kendi ilinin kadın verisi	Veri girer, kendi ilinde birimler arası karşılaştırma yapar
İl Koordinatörü (E)	Sadece erkek kolu	Kendi ilinin erkek verisi	Veri girer, kendi ilinde birimler arası karşılaştırma yapar
Bölge Koordinatörü	Cinsiyet fark etmez	Bölgesindeki tüm illerin HEM K HEM E verisi	Rapor, karşılaştırma (iller, cinsiyetler, birimler), analiz
Genel Merkez Birim Başkanı	Cinsiyet fark etmez	Tüm illerin sadece kendi biriminin (ör. sadece üniversite) verisi	Birim odaklı ülke geneli analiz ve karşılaştırma
Genel Merkez Admin	Tam yetki	Her şey	Her şey + karne oluşturma + AI raporlaması + sistem yönetimi
1.2 Cinsiyet Görünüm Mantığı (Kritik Kural)
Varsayılan = Birleşik
Sistem her ekranı varsayılan olarak K+E TOPLAMI şeklinde gösterir.
Kullanıcı sadece filtre uyguladığında ayrım yapılır (Kadın/Erkek/Her İkisi).
İl koordinatörü için filtre kilitli — sadece kendi kolunu görür (RLS bunu sağlar).
Bölge ve üstü için filtre serbest — istediği kombinasyonu görebilir.
1.3 Kim Neyi Karşılaştırabilir?
Rol	Karşılaştırma Yetkisi
İl Koordinatörü	Kendi ilinde birimler arası (üniversite vs lise vs ortaokul vs çocuk). Dönemler arası (bu ay vs geçen ay).
Bölge Koordinatörü	İller arası (kendi bölgesinde). Cinsiyet kolları arası. Birimler arası. Dönemler arası.
Genel Merkez Birim Başkanı	Tüm illerin kendi biriminde: iller arası, bölgeler arası, dönemler arası, cinsiyet kolları arası.
Admin	Her boyutta, her kombinasyonda.
 2. Kalıcılık ve Arşiv Sistemi
Silme Politikası
Hiçbir kayıt otomatik silinmez.
Üretilen her rapor, karne, AI analizi ve PDF çıktısı üretim tarihiyle sisteme kaydedilir.
Kullanıcı arşivden istediğinde eski bir raporu açıp görebilir.
Yalnızca yetkili kullanıcı elle sil butonuna basarsa kayıt silinir (soft delete önerilir — geri getirilebilir).

2.2 /arsiv Ekranı
•	Tüm üretilmiş PDF'ler ve AI analizleri kronolojik listede.
•	Filtre: tarih aralığı, il, tür (haftalık rapor / karne / analiz), üreten kullanıcı.
•	Her satır: küçük önizleme + "PDF olarak aç" + "AI yorumunu göster" + (yetkiliyse) "Sil".
•	Silinen kayıt 30 gün soft-delete olarak durur, sonra kalıcı silinir (istenirse hiç silinmez de yapılabilir).

// prisma/schema.prisma
 
model User {
  id            String    @id @default(cuid())
  email         String    @unique
  passwordHash  String
  fullName      String
  role          Role
  genderBranch  GenderBranch?  // NULL = cinsiyet bağımsız (bölge, birim başkanı, admin)
  provinceId    Int?
  regionId      Int?
  unitId        Int?          // sadece Genel Merkez Birim Başkanı için
  isActive      Boolean   @default(true)
  deactivatedAt DateTime?
  province      Province? @relation(fields: [provinceId], references: [id])
  region        Region?   @relation(fields: [regionId], references: [id])
  unit          Unit?     @relation(fields: [unitId], references: [id])
  activities    Activity[]
}
 
enum Role {
  IL_KOORDINATOR
  BOLGE_KOORDINATOR
  MERKEZ_BIRIM_BASKANI
  ADMIN
}
 
enum GenderBranch {
  K
  E
}
 
model Activity {
  id               String        @id @default(cuid())
  periodId         Int
  institutionId    Int
  facultyId        Int?
  activityTypeId   Int
  participantCount Int
  genderBranch     GenderBranch  // her kayıtta zorunlu
  location         String?
  note             String?
  createdBy        String
  createdAt        DateTime      @default(now())
  deletedAt        DateTime?     // soft delete
  
  period       Period       @relation(fields: [periodId], references: [id])
  institution  Institution  @relation(fields: [institutionId], references: [id])
  faculty      Faculty?     @relation(fields: [facultyId], references: [id])
  activityType ActivityType @relation(fields: [activityTypeId], references: [id])
  user         User         @relation(fields: [createdBy], references: [id])
  
  @@index([periodId, institutionId])
  @@index([genderBranch])
}
 5. Yetki Kontrol Merkezi (Uygulama Kodunda)
Yetki kuralları tek bir yerde toplanır: lib/authz.ts. Tüm API route'lar bu fonksiyonları çağırır. Bu, tutarlılık ve test edilebilirlik için kritiktir.
// lib/authz.ts
 
import { Role, GenderBranch } from '@prisma/client'
 
export type SessionUser = {
  id: string
  role: Role
  genderBranch: GenderBranch | null
  provinceId: number | null
  regionId: number | null
  unitId: number | null
}
 
// activities için Prisma where clause üretir
export function buildActivityFilter(
  user: SessionUser,
  requestedGenderFilter?: GenderBranch | 'ALL'
) {
  const where: any = { deletedAt: null }
  
  switch (user.role) {
    case 'IL_KOORDINATOR':
      // Kendi ili + kendi cinsiyet kolu (kilitli)
      where.institution = { provinceId: user.provinceId }
      where.genderBranch = user.genderBranch
      break
    
    case 'BOLGE_KOORDINATOR':
      // Bölgesindeki tüm iller; cinsiyet: kullanıcının seçtiği filtreye göre
      where.institution = { province: { regionId: user.regionId } }
      if (requestedGenderFilter && requestedGenderFilter !== 'ALL') {
        where.genderBranch = requestedGenderFilter
      }
      break
    
    case 'MERKEZ_BIRIM_BASKANI':
      // Tüm iller; sadece kendi birimi; cinsiyet: filtreye göre
      where.institution = { unitId: user.unitId }
      if (requestedGenderFilter && requestedGenderFilter !== 'ALL') {
        where.genderBranch = requestedGenderFilter
      }
      break
    
    case 'ADMIN':
      // Her şey; cinsiyet: filtreye göre
      if (requestedGenderFilter && requestedGenderFilter !== 'ALL') {
        where.genderBranch = requestedGenderFilter
      }
      break
  }
  
  return where
}
 
// Kullanıcı bir eylemi yapabilir mi?
export function canWrite(user: SessionUser): boolean {
  return user.role === 'IL_KOORDINATOR' || user.role === 'ADMIN'
}
 
export function canGenerateReport(user: SessionUser): boolean {
  return user.role !== 'IL_KOORDINATOR'  // il koordinatörü rapor üretmez, sadece girer
}
 
export function canDelete(user: SessionUser): boolean {
  return user.role === 'ADMIN'
}
Prensip
Cinsiyet için varsayılan 'ALL' (birleşik). Kullanıcı filtre uygularsa 'K' veya 'E' geçer.
İl koordinatörü için requestedGenderFilter yok sayılır; kendi kolu zorunlu.
Tüm sorgular bu fonksiyondan geçer — hiçbir API route doğrudan Prisma where yazmaz.

8.	Kullanıcıya indirme linki veya "Arşive kaydedildi" mesajı döner.
7.3 Arşiv Kalıcılığı
Kesin Kural
Otomatik silinme YOK. reports.deletedAt varsayılan NULL.
Yalnız Admin "Sil" butonuna basarsa deletedAt = now() yazılır (soft delete).
Soft delete edilenler /arsiv'de görünmez ama audit_log'da kaydı kalır.
İstenirse Admin "Kalıcı sil" butonu ile fiziksel dosyayı da diskten kaldırabilir.

 5. Yetki Kontrol Merkezi (Uygulama Kodunda)
Yetki kuralları tek bir yerde toplanır: lib/authz.ts. Tüm API route'lar bu fonksiyonları çağırır. Bu, tutarlılık ve test edilebilirlik için kritiktir.
// lib/authz.ts
 
import { Role, GenderBranch } from '@prisma/client'
 
export type SessionUser = {
  id: string
  role: Role
  genderBranch: GenderBranch | null
  provinceId: number | null
  regionId: number | null
  unitId: number | null
}
 
// activities için Prisma where clause üretir
export function buildActivityFilter(
  user: SessionUser,
  requestedGenderFilter?: GenderBranch | 'ALL'
) {
  const where: any = { deletedAt: null }
  
  switch (user.role) {
    case 'IL_KOORDINATOR':
      // Kendi ili + kendi cinsiyet kolu (kilitli)
      where.institution = { provinceId: user.provinceId }
      where.genderBranch = user.genderBranch
      break
    
    case 'BOLGE_KOORDINATOR':
      // Bölgesindeki tüm iller; cinsiyet: kullanıcının seçtiği filtreye göre
      where.institution = { province: { regionId: user.regionId } }
      if (requestedGenderFilter && requestedGenderFilter !== 'ALL') {
        where.genderBranch = requestedGenderFilter
      }
      break
    
    case 'MERKEZ_BIRIM_BASKANI':
      // Tüm iller; sadece kendi birimi; cinsiyet: filtreye göre
      where.institution = { unitId: user.unitId }
      if (requestedGenderFilter && requestedGenderFilter !== 'ALL') {
        where.genderBranch = requestedGenderFilter
      }
      break
    
    case 'ADMIN':
      // Her şey; cinsiyet: filtreye göre
      if (requestedGenderFilter && requestedGenderFilter !== 'ALL') {
        where.genderBranch = requestedGenderFilter
      }
      break
  }
  
  return where
}
 
// Kullanıcı bir eylemi yapabilir mi?
export function canWrite(user: SessionUser): boolean {
  return user.role === 'IL_KOORDINATOR' || user.role === 'ADMIN'
}
 
export function canGenerateReport(user: SessionUser): boolean {
  return user.role !== 'IL_KOORDINATOR'  // il koordinatörü rapor üretmez, sadece girer
}
 
export function canDelete(user: SessionUser): boolean {
  return user.role === 'ADMIN'
}
Prensip
Cinsiyet için varsayılan 'ALL' (birleşik). Kullanıcı filtre uygularsa 'K' veya 'E' geçer.
İl koordinatörü için requestedGenderFilter yok sayılır; kendi kolu zorunlu.
Tüm sorgular bu fonksiyondan geçer — hiçbir API route doğrudan Prisma where yazmaz.

 7. PDF Üretim ve Arşiv Sistemi
7.1 Üretilen PDF Türleri
Tür	İçerik	Ne Zaman
Haftalık İl Raporu	Bir ilin bir haftadaki tüm faaliyetleri + AI özet + grafik	Kullanıcı butonla üretir
İl Karnesi	Bir ilin yıllık karne dökümü (sunumdaki mockup gibi) + AI yorum	Yıl sonu veya istenildiğinde
Bölge Raporu	Bir bölgenin dönem özeti	Bölge koordinatörü isterse
Ülke Karnesi	Tüm iller sıralaması + öne çıkanlar + AI yorum	Yıl sonu Türkiye teşkilat kampı için

7.2 PDF Üretim Akışı
1.	Kullanıcı ekranda "PDF olarak indir" veya "Bu raporu arşive kaydet" butonuna tıklar.
2.	API route /api/reports/generate çağrılır.
3.	Sunucu tarafında ilgili veriyi Prisma ile çeker.
4.	İsteğe göre OpenRouter API'den AI yorumu alır.
5.	React bileşeni olarak PDF üretilir (@react-pdf/renderer).
6.	Dosya /storage/reports/{yıl}/{ay}/{uuid}.pdf yoluna kaydedilir.
7.	reports tablosuna meta bilgi eklenir (tip, kapsam, dönem, dosya yolu, üreten kullanıcı, üretim tarihi).
8.	Kullanıcıya indirme linki veya "Arşive kaydedildi" mesajı döner.
7.3 Arşiv Kalıcılığı
Kesin Kural
Otomatik silinme YOK. reports.deletedAt varsayılan NULL.
Yalnız Admin "Sil" butonuna basarsa deletedAt = now() yazılır (soft delete).
Soft delete edilenler /arsiv'de görünmez ama audit_log'da kaydı kalır.
İstenirse Admin "Kalıcı sil" butonu ile fiziksel dosyayı da diskten kaldırabilir.

   - Çocuk

Kurallar:
- Kurumsal, tarafsız ton
- Sadece verideki bilgi
- Cinsiyet kolu filtre uygulanmadıysa toplam ver; uygulandıysa belirt

- Karşılaştırılan boyutları (dönem/cinsiyet/il vs.) açıkça belirt
- Yargı yerine somut sayı ile konuş
- En fazla 90 kelime, düz metin
D) Trend Yorumu

Görevin:
- Genel yönü belirt (yükseliş / düşüş / durgun)

Kurallar:
- Panik yaratma, doğal ton
- Mevsimsel etkileri (yaz tatili, sınav dönemi) not düşebilirsin
- En fazla 80 kelime
E) İl Birim Yönlendirmesi (İl koordinatörüne özel)

Görevin:
- En yüksek performanslı birim ve en düşük performanslı birimi belirt
- Yüksek performanslı birime dair 1 tebrik / sürdürme önerisi
- Düşük performanslı birime dair 1 somut aksiyon önerisi
  ("lise çalışmalarınıza ağırlık verin" gibi)
- İl bazlı özel bir bilgi biliyorsan (ör. büyük üniversitesi varsa)
  belirtme; sadece verilen veriyle konuş
 
Kurallar:
- Motive edici, arkadaş canlısı ama profesyonel ton
- Cinsiyet kolunu "kadın kolu" veya "erkek kolu" olarak açıkça belirt
- 80-100 kelime, düz metin
F) Serbest Soru
Sen GENÇ İHH raporlama sisteminin veri asistanısın. Kullanıcı doğal
dilde soru soruyor; sana ilgili yapılandırılmış veri ve soruyu
veriyorum.
 
Görevin:
- Soruyu veriye dayanarak cevapla
- Cevaba en az bir somut sayı koy
- Veri yetersizse dürüstçe belirt; uydurma
 
Kurallar:
- Kurumsal, saygılı ton
- Kullanıcının yetkisi dışındaki veri gelmez; gelmediyse belirtme
- En fazla 120 kelime
